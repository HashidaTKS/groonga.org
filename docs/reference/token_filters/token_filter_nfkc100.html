7.9.2.1. TokenFilterNFKC100¶

7.9.2.1.1. Summary¶

This token filter can translate a token for katakana to hiragana with same option of NormalizerNFKC100. This token filter convenient when you want to get reading of token as hiragana.

7.9.2.1.2. Syntax¶

TokenFilterNFKC100 has a parameter:

TokenFilterNFKC100("unify_kana", true)

7.9.2.1.3. Usage¶

7.9.2.1.4. Simple usage¶

Here is an example of TokenFilterNFKC100. リンゴ is translated to りんご.

Execution example:

tokenize TokenDelimit "リンゴ"   --token_filters 'TokenFilterNFKC100("unify_kana", true)'
# [
#   [
#     0,
#     1545901643.191951,
#     0.0003898143768310547
#   ],
#   [
#     {
#       "value": "りんご",
#       "position": 0,
#       "force_prefix": false,
#       "force_prefix_search": false
#     }
#   ]
# ]

TokenFilterNFKC100 is not translate a token of hiragana and kanji as below.

Execution example:

tokenize TokenDelimit "りんご 林檎"   --token_filters 'TokenFilterNFKC100("unify_kana", true)'
# [
#   [
#     0,
#     1545901725.412921,
#     0.0003237724304199219
#   ],
#   [
#     {
#       "value": "りんご",
#       "position": 0,
#       "force_prefix": false,
#       "force_prefix_search": false
#     },
#     {
#       "value": "林檎",
#       "position": 1,
#       "force_prefix": false,
#       "force_prefix_search": false
#     }
#   ]
# ]

7.9.2.1.5. Advanced usage¶

You can output all input string as hiragana with cimbining TokenFilterNFKC100 with use_reading option of TokenMecab as below.

Execution example:

7.9.2.1.6. Parameters¶

7.9.2.1.6.1. Required parameters¶

There are a required parameters unify_kana.

7.9.2.1.6.1.1. unify_kana¶

Translate a token katakana to hiragana.
