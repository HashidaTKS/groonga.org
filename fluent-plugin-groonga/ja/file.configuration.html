設定

fluent-plugin-groongaは2つのfluentdプラグインを提供します。1つが groonga inputプラグインで、もう1つが groonga outputプラグインです。このドキュメントではこれらのプラグインの設定パラメーターを説明します。

groonga inputプラグイン

利用可能なパラメーターは以下の通りです。

  • protocol: groongaのコマンドを受け取るプロトコルを指定します。
    • 利用可能な値: http と gqtp
    • デフォルト: http
  • bind: バインドするアドレスを指定します。
    • デフォルト: 0.0.0.0
  • port: ポート番号を指定します。
    • デフォルト: 10041
  • real_host: 本当のgroongaサーバーのアドレスを指定します。必須です。
    • デフォルト: デフォルトはなし。
  • real_port: 本当のgroongaサーバーのポート番号を指定します。
    • デフォルト: 10041
  • emit_commands: TODO

以下は設定例です。

<source>
  type groonga
  protocol http
  bind 127.0.0.1
  port 10041
  real_host 192.168.0.1
  real_port 10041
</source>

groonga outputプラグイン

  • protocol: groongaのコマンドをgroongaへ送るときに使うプロトコルを指定します。
    • 利用可能な値: http と gqtp と command
    • デフォルト: http
  • http あるいは gqtp を使った場合:
    • host: groongaサーバーのアドレスを指定します。
      • デフォルト: localhost
    • port: groongaサーバーのポート番号を指定します。
      • デフォルト: 10041
  • command を使った場合:
    • groonga: groongaコマンドのパスを指定します。
      • デフォルト: groonga
    • database: groongaデータベースのパスを指定します。必須です。
      • デフォルト: デフォルトなし。
    • arguments: groongaコマンドに追加する引数を指定します。
      • デフォルト: 追加の引数はありません。

以下は設定例です。

<match groonga.command.*>
  type groonga
  protocol command
  database /tmp/groonga/db
</match>
